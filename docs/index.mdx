Naked UI provides interaction, focus, keyboard, and accessibility behavior
without prescribing presentation. Its builders expose immutable state snapshots,
so your design system owns every pixel while the control keeps a predictable
behavioral contract.

## Requirements

- Flutter 3.41 or newer
- Dart 3.11 or newer

## Install

```shell
flutter pub add naked_ui
```

Then import the public library:

```dart
import 'package:naked_ui/naked_ui.dart';
```

## First control

```dart
import 'package:flutter/material.dart';
import 'package:naked_ui/naked_ui.dart';

class SubmitButton extends StatelessWidget {
  const SubmitButton({super.key, required this.onPressed});

  final VoidCallback? onPressed;

  @override
  Widget build(BuildContext context) {
    return NakedButton(
      onPressed: onPressed,
      builder: (context, state, child) {
        final color = state.when(
          disabled: Colors.grey.shade300,
          pressed: Colors.blue.shade800,
          hovered: Colors.blue.shade600,
          focused: Colors.blue.shade400,
          orElse: Colors.blue,
        );

        return AnimatedContainer(
          duration: const Duration(milliseconds: 120),
          padding: const EdgeInsets.symmetric(horizontal: 16, vertical: 12),
          decoration: BoxDecoration(
            color: color,
            borderRadius: BorderRadius.circular(8),
            border: state.isFocused
                ? Border.all(color: Colors.white, width: 2)
                : null,
          ),
          child: const Text('Submit'),
        );
      },
    );
  }
}
```

`NakedButton` owns activation, focus, pointer, keyboard, feedback, and button
semantics. The builder owns appearance. A null `onPressed` disables activation
and exposes the disabled state to both the builder and assistive technology.

## State snapshots

Every builder receives a component-specific subclass of `NakedState`. Common
state is available through:

- `states`, an immutable `Set<WidgetState>`
- `isHovered`, `isFocused`, `isPressed`, `isDragged`, `isSelected`, and
  `isDisabled`
- `when(...)`, which resolves branches in a documented priority order
- `whenOrNull(...)`, when no fallback is needed

Component snapshots add domain state such as `NakedSliderState.value`,
`NakedTabState.tabId`, or `NakedCheckboxState.isChecked`.

## Accessibility contract

Naked controls provide their native role, state, and actions. You remain
responsible for content-specific accessible names and hints:

- Prefer visible text that naturally labels the control.
- Provide `semanticLabel` for icon-only or otherwise unlabeled controls.
- For non-string select values, provide `semanticValue` or
  `semanticValueFormatter`.
- Keep visible focus indicators and sufficiently large pointer targets in your
  presentation layer.
- Use `excludeSemantics` only when replacing the Naked UI semantics contract
  yourself. Descendant semantics are preserved.

## Choose a component

### Actions and choices

- [Button](/widget/button) — invoke an action
- [Toggle](/widget/toggle) — binary toggle button or switch
- [Checkbox](/widget/checkbox) — independent boolean or tristate choice
- [Radio](/widget/radio) — one choice in a `RadioGroup`

### Input

- [Text field](/widget/textfield) — native text editing with custom chrome
- [Select](/widget/select) — choose one value from an anchored overlay
- [Slider](/widget/slider) — continuous or discrete numeric input

### Structure and overlays

- [Tabs](/widget/tabs) — roving-focus tab list and panels
- [Accordion](/widget/accordion) — constrained expandable sections
- [Menu](/widget/menu) — anchored action menu
- [Dialog](/widget/dialog) — modal route and dialog semantics
- [Tooltip](/widget/tooltip) — hover/touch hint
- [Popover](/widget/popover) — arbitrary anchored content

## Ownership and lifecycle

Controllers and focus nodes supplied by the caller remain caller-owned and must
be disposed by the caller when their API has a disposal lifecycle. Internally
created objects are owned by the component. Controlled values remain the source
of truth; callbacks request changes rather than mutating caller state.

Browse the component guides for complete examples and API-specific semantics.
